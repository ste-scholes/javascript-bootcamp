Information

All HTML objects consist little boxes, all these have padding, margin, border

Padding is space round the text

Border is the border around the object

Margin is the space outside the border, that surrounds the padding

Cascading

Styles work by cascading the attributes down through the document

A style in the body tag will be overwritten by a style that is more specific

Think of this as a style closer to the object has presidence over one further up the chain

Overriding styles

Which style overides the other depends on

The order they appear within the in page style is not important

The folowing h1 has two style attributes assigned, pink text and blue text

h1 class="pink blue"

What does matter is the order that the styles are set within the style sheet - If pink is after blue then it will override it no matter what

Overriding Styles with ID

IDs are more specific than other selectors

If orange was written at the top of the style sheet, and pink and blue came after you would assume they would take presidence, this is NOT true

h1 class="pink blue" id="orange"

IDs are more specific as there is only supposed to be one per attribute, classes appear on many

Inline styles

Inline styles are more specific than ID as they appear actually on the element

Important

!important

This means the style is the most important, this when applied will not be overwritten further down the chain, it will ignore ID and inline styles

CSS variables

You can assign things like colour to variables, then when you change them in the variable they will be picked up throughout the document

Example

within the brackets of a selector .penguin {} the colours are declared as variables

--penguin-skin: black;

--penguin-belly: gray;

--penguin-beak: yellow;

They are then called by the variable name throughout the document

background: var(--penguin-skin, gray); // the gray is the default backup

Declaring variables in Root

This is the same as making a variable with global scope, it will be avaiable througout the entire document

Example

:root{ }
