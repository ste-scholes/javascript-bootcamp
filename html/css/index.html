Position Statc - Default

Static is the default position of HTML elements added to a page

Use the following to change how elements react

Static elements cannot have elements fixed absolutely to them

Position - Relative

Relative is removed from the document flow - this causes it to overlap rather than push other elements down

This isn't something we would usually use with top, right, left and bottom as it is difficult to style a document that has elements removed from it in this way

.child-one {
  position: relative;
  left: 10px;
  top: 10px;
}

Position - Absolute

Using position absolute is useful for positioning something in an exact location, where you don't want it to move.

It is essenially removed from the flow of the document, but can bepositioned using top, right, left and bottom

Absolute positioning positions the object absolutly relative to it's parent container

REMEMBER HTML elements are by default static, that means you cannot fix another element absolutly to them. You need to useful relative, absolute, sticky, or fixed in order to make an element fix absolutely inside it.

.parent {
  position: relative;
}

.child-one {
  position: absolute;
  top: 10px;
}

Position - Fixed

Fixed position elements ignore all parent containers

They also stay in the same position, even when the page is scrolled

In the example code below, the child element one, stays in the position set in the code - 10px from the top, zero px from the left. It will retain this position regardless of scoling

.parent {
  position: relative;
  height: 200vh;
}

.child-one {
  position: fixed;
  top: 10px;
  left: 0;
}

Position Sticky

Sticky starts off rlative until it hits the top of the page where it becomes fixed if top: 0px as in example code

This is used where you see menu bars scroll up and then stick at the top of the page.

.parent {
  position: relative;
  height: 200vh;
}

.child-one {
  position: sticky;
  top: 0px;
}
