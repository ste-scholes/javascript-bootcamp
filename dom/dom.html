getElementById

document.getElementById

Will only get the first ID that matches back

There should only be one match, but if there is more, it will return the first one it encounters

const img = document.getElementById('winter')

console.dir(img)

This returns what at first looks like HTML but it is in fact an object that contains all the information about the attribute

The above is classed as an HTMLImageElement

There are different types of HTML elements

getElementsbyTagName

getElementsByTagName('li')

Returns a collection containing all elements of that type on the page, even if there is only one

You can get all the images, li, h2, etc on a page.

The results looks like an array, but it is not an array, although you can manipulate it like an array to an extent

Think of the arguments object, inside a function there is an array like object that is not an array, this is similar

const elements = document.getElementsByTagName('input');
console.dir(elements);
console.log(elements[1]) // returns: input type="password" placeholder="password"

The second item of the inputs


getElementsByClassName

document.getElementsByClassName()

This like the above tries to bring back more than one element and adds to a collection

Once again the results look like an array without being able to use most array methods

Examples

const header = document.getElementsByClassName('header');

console.log(header)

Returns HTMLCollection [h1.header]

const list = document.getElementsByClassName('special')

console.log(list)

Returns HTMLCollection(2) [li.special, li.special]

Seach for items embedded within other items

In the below the class special has been used on p tags as well as within an ul

How do we choose just the li tag that has a class of special?

// below returns all with class special

const special = document.getElementsByClassName('special');

console.log(special)

// if I want to isolate the li tag - the below means [array] index 0 or the first element

const ul = document.getElementsByTagName('ul')[0]

console.log(ul)

// ul brings back the ul that now has it's own version of getElementsByClassName that can be searched for 'special'

const specialLi = ul.getElementsByClassName('special')

console.log(specialLi)

querySelector

A newer, all-in-one method to select a single element

Pass in a css selector

Remeber querySelector only returns a single instance

Example

Finds fist h1 element:

document.querySelector('h1')

Finds first element with ID of red:

document.querySelector('#red')

Finds first element with class of:

document.querySelector('.big')

Remember you need to be specific! .class #main element (h1) etc

Example

Select a password field on page1 form

const psw = document.querySelector('input[type="password"]')

console.log(psw)

This is the same selector that is used in CSS - input[type="password"]

querySelectorAll

Uses the exact same syntax but returns a collection (all instances)

Examples

const form = document.querySelectorAll('input')

console.log(form)

returns NodeList(3) [input, input, input]

const specialAttr = document.querySelectorAll('.special');

console.log(specialAttr)

returns NodeList(4) [p.special, li.special, li.special, li.special]

querySelectorAll returns a node list, this is similar to the array like object returned by getElementsByTagName, it is just held in another type of container

Examples

const specialAttr = document.querySelectorAll('.special');

console.log(specialAttr)returns all items with class of special

const arr2 = [...specialAttr]returns eveything colected with class of special and spreads it into an array

console.log(arr2)returns the array created by the ... spread above

console.log(arr2.length)returns the number of items in the array (4)

console.log(arr2[1]) returns the element in the second array position
