DOM

Document Object Model

The document object is our entry point into the world of the DOM. It contains representations of all the content on a page, plus lots of useful methods and propeties

It is an hierarchy or elements and their children

document

Everything is nested within the document

getElementById

document.getElementById

Will only get the first ID that matches back

There should only be one match, but if there is more, it will return the first one it encounters

const img = document.getElementById('winter')

console.dir(img)

This returns what at first looks like HTML but it is in fact an object that contains all the information about the attribute

The above is classed as an HTMLImageElement

There are different types of HTML elements

getElementsbyTagName

getElementsByTagName('li')

Returns a collection containing all elements of that type on the page, even if there is only one

You can get all the images, li, h2, etc on a page.

The results looks like an array, but it is not an array, although you can manipulate it like an array to an extent

Think of the arguments object, inside a function there is an array like object that is not an array, this is similar

const elements = document.getElementsByTagName('input');
console.dir(elements);
console.log(elements[1]) // returns: input type="password" placeholder="password"
