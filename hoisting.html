Hoisting

Hoisting Var

Variables without a value are set to undefined

Example

var animal = 'Tapier';
console.log(animal); // Returns Tapier

However

console.log(animal);
var animal = "Tapir"; // Returns undefined

This happens because of Hoisting

JavaScript hoists the declaration part: var animal

Explanation

JavaScript runs the code in this order

  1. var animal

  2. console.log(animal)

  3. animal = 'Tapir'

JavaScript hoists the var animal declaration, it then console.log(animal), finally it assigns the value Tapir

To avoid Hoisting always delare and initialize values before you use them

Hoisting Let

Variable declaration made with let are not hoisted

Example

console.log(shrimp)
let shrimp = 'Harelquin Shrimp';

The above code brings back following error:

Uncaught ReferenceError: Cannot access 'shrimp' before initialization

This is one reason to use let instead of var

Hoisting Const

Just like let, const is not hoisted and will return an error if you try to use it before it has been delcared and initialized

Hoisting - Function Declrations

Function declarations can be called before the function

Example

howl(); // calling the function first

function howl() {
  console.log('Awwwooooooo!')
}

In the above case you can think of them as being put at the top of your file

Hoisting - Function Expression

Setting the function to a variable will not hoist the declaration

The function hoot exisits but it is set to undefined

hoot()

var hoot = function(){
console.log('Wit Woooo')
}

The above code does not hoist the function expression it returns the following error

Uncaught TypeError: hoot is not a function

Using let will return the same error as before

Uncaught ReferenceError: Cannot access 'hoot' before initialization

const will return the same error as var

Uncaught TypeError: hoot is not a function
