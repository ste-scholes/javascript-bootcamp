Destructuring Function Params

In the following code example we have a function that accepts two parameters and multiplies them

function multi(x,y) {
return x * y;
}

console.log(multi(2,5)) // returns 10

But what happens if we only input one parameter?

console.log(multi(2,)) // returns NaN

y is set to undefined as there was no paramter inputted

Default - old way

You could set a default paramter in case one was not inputted

function mult(x,y){
 if(typeof y ==='undefined') { //This checked for a value
  y = 1; //If no value detected y is set to 1
  }
  return x * y;
}

console.log(mult(5,2))

If a value is detected then the sum ignores the if statement and carries on

This uses typeof which returns what type of primative 'string', number etc.

Ternary example

The same as above but as a ternary operator

function multip(x,y) {
 y = typeof y === 'undefined' ? 1 : y
  return x * y;
}

console.log(multip(2,6))

Default Params - The New Way

The new way to set a default parameter is to set the value within the parameters

function multiply(x, y = 1) { //Here we can see y default set to 1
  return x * y
}

console.log(multiply(4,9))
