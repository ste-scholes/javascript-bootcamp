Functions in detail

Scope

Variable visiability

The location where a variable is defined dicatets where we have access to that variable.

Function Scope

Variable defined within a variable, it is not acessable outside the scope of the variable

Block Scope

Variable created within a block are scoped to that block and are not avaiable outside

Example

if(true){
  let animal = 'eel'
}

console.log(animal) // brings back error animal not defined - it is out of scope

Scope Example

function doubleArr(arr) {
  const result=[]; // outside block scope
  for (let num of arr) {
  let double = num * 2; // within block scope
  result.push(double)
}
 return result // outside block scope
}

console.log(doubleArr([1,2,3]))

Var

Let and Const have different scoping rules to VAR

Var defies block scoping, a variable defined by var is available both inside and outside of a block scope

This is different to other programming languages

This is why let and const were introduced

var can cause problems especially in loops where i could be accessed outside, there are other reason that var is a problem, you could overwrite one var with another or the same name and so on

Lexical Scope

Nested Functions are lexically bound to the scope of their parent/outer function

If you have a function wihin a function

Without calling the inner function you will not get access to it

It is a way way relstionshsip, you can access the child via the parent but not the other way round

Example

In the below example movie is not called by the inner function but is available via the outer function

function outer(){
  let movie = 'Star Wars';
  function inner() {
  console.log(movie.toUpperCase())
}
  inner() //calls inner function
}

console.log(outer())

If you try to log inner, it will bring back an erros, because it is out of scope and never declared

function out() {
let movie = 'Ghostbusters'

function inside() {
